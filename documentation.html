HTML Structure

Mint has a very well commented and intuitive HTML5 markup which can be easily customized. Comments and text here will explain better what you need to use in order to make a new design. Here's the basic markup for a page:

The Head

The head tag contains the title, meta tags and CSS links which you will use in your pages.

<head>

    <title>Mint - Your Title goes here</title>

    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">

    <meta name="viewport" content="width=device-width, height=device-height, initial-scale=1, maximum-scale=1, user-scalable=no">

    <meta name="description" content="Your description goes here. ">

    <link rel="author" href="http://grozav.com"/>

    <!-- Core CSS - Include with every page -->
    <link href="css/bootstrap.css" rel="stylesheet">
    <link href="http://netdna.bootstrapcdn.com/font-awesome/4.0.3/css/font-awesome.min.css" rel="stylesheet">

    <!-- Page-Level Plugin CSS - Blank -->

    <!-- Mint Admin CSS - Include with every page -->
    <link href="css/mint-admin.css" rel="stylesheet">

</head>

When copying elements from another page, make sure to copy the CSS file links related to them as well.

As you can see, pages contain 3 types of css files: Core CSS, Page-level Plugin CSS and Mint Admin CSS.

  • The Core CSS is the Bootstrap and FontAwesome CSS files which are the dependencies of the CSS classes we use in our markup.
  • The Page-level Plugin CSS are the files you need in order to make the plugins used look the way they're supposed to.
  • The Mint Admin CSS is a CSS file which contains custom styling elements specific to Mint Admin.

The Body

The body tag contains all of our HTML elements and scripts which you will see in your pages.

<body>
      <div id="wrapper">
            <!-- Top Nav -->
            <nav class="navbar navbar-default navbar-static-top" role="navigation"></nav>
            <!-- Side Nav -->
            <nav class="navbar-default navbar-static-side" role="navigation"></nav>
            <!-- Main Content -->
            <div id="page-wrapper"></div>
      </div>
      <script></script>
</body>

In Mint, we have active links and active list items. An active link is the one that shows the page you're currently viewing. An active list item shows that the active link is inside the list dropdown, therefore it will open the dropdown on page load.

To set a link or list item active, simply add the active class to it. Here's an example:

<!-- Shows that the dropdown contains the current page link -->
<li class='active'> 
    <!-- Shows that the dropdown contains the current page link if the dropdown gets closed by the user -->
    <a class='active' href="#"><i class="fa fa-sitemap fa-fw fa-3x"></i> Multi-Level Dropdown<span class="fa arrow"></span></a>
    <ul class="nav nav-second-level">
        <li>
            <a href="#">Second Level Item</a>
        </li>
        <li>
            <!-- Shows the current page link -->
            <a class='active' href="#">Current Page</a>
        </li>
    </ul>
</li>

Scripts

In order to make the page load faster, we include the scripts at the bottom of our page.

<!-- Core Scripts - Include with every page -->
<script src="js/jquery-1.10.2.js"></script>
<script src="js/bootstrap.min.js"></script>
<script src="js/plugins/metisMenu/jquery.metisMenu.js"></script>

<!-- Page-Level Plugin Scripts - Blank -->

<!-- Mint Admin Scripts - Include with every page -->
<script src="js/mint-admin.js"></script>

<!-- Page-Level Demo Scripts - Blank - Use for reference -->

When copying elements from another page, make sure to copy the JS scripts related to them as well. Also check out the demo pages and use them to create your custom graphs.

Let's see what each type of script is:

  • Core Scripts are the ones needed on every page. These are dependencies that make the jQuery, Bootstrap and Menu Interface UI parts work.
  • Page-Level Plugin Scripts this is where you include scripts needed for elements such as dataTables and graphs.
  • Mint Admin Script is the main javascript files which makes the template run properly.
  • Page-Level Demo Scripts are for some plugins, like the graphs, which need to be initialized properly.

CSS Structure

SASS is strongly recommended for customizing the template since it reduces to work to changing a few intuitive variables.

SASS

The head tag contains the title, meta tags and CSS file links which you will use in your pages.

We have 3 main SASS files:

  • bootstrap.scss - which generates our custom bootstrap 3 ui
  • css3-mixins.scss - which provides our css3 mixins
  • mint-admin.scss - which generates the css design which is not default in bootstrap

Bootstrap

Let's analyze the bootstrap.scss file. The main file you will want to edit will be the _variables.scss which contains the variables (dimensions, variables, colors) used by all the other bootstrap files. Let's see the content of our _variables.scss file that you should focus on:

Theme Color

You can change the color of the whole admin template by simply changing the global variable $theme-color. 

//theme default color, can be also set to #HEXCOD
$theme-color: nth($color-value, 1);
//theme content background 
$wrapper-bg: #e7e7e7; 
//theme content elements background
$wrapper-default: #f5f5f5; 
//theme content elements light background
$wrapper-light: #fbfbfb;

Graphs color can be changed inside the javascript demo files related to them.


Themes

In Mint version 1.1, we have implemented color themes, therefore you do not have to worry about your favorite color. Mint comes with 20 Color Themes implemented for you to use.

A theme is simply implemented by replacing the CSS file link inside the <head> tag. 

<link href="css/mint-admin.css" rel="stylesheet">
<link href="css/bootstrap.css" rel="stylesheet">

Replace the links above with a chosen theme css, where [color] is a color chosen from the list below. 

<link href="css/themes/mint-admin-[color].css" rel="stylesheet">
<link href="css/themes/bootstrap-[color].css" rel="stylesheet">

JavaScript

Mint uses several javascript plugins, each of them coming with a complete demo page and a link to their documentation.

There's not really much to say here. The only javascript our admin system uses besides the plugins is a resize bind to make the charts responsive. 

var morris = new Morris(data);

var delay = (function() {
    var timer = 0;
    return function(callback, ms) {
        clearTimeout(timer);
        timer = setTimeout(callback, ms);
    };
})();
$(window).resize(function() {
    delay(function() {
        morris.redraw();
    }, 500);
}).trigger('resize');

We use the timeout trick to make the graphs resize once the resize is done, since the redraw operation is quite resource costly. I recommend using it whenever you have something to resize.
